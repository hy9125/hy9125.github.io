      HADOOP配置2.X

      Post on 2016-07-20

      配置HADOOP环境2.X（完全分布式）

      在此之前应该干的事（4台虚拟机，配置好IP，配置好JAVA）
      最近1.X和2.X有点混。又开始给自己挖坑了
      向度娘和必应致敬！

      固定IP

      • 这一点非常重要，是实现SSH免密钥登陆，加载映射关系的基础
      • 例如；
        • master 192.168.166.11
        • node1 192.168.166.22
        • node2 192.168.166.33

      配置SSH

      • 安装SSH和VIM（也可以用自带的VI）
        • sudo apt-get install vim
        • sudo apt-get install ssh
      • 客户端用PUTTY实现SSH登陆，使用WINSCP传送文件

      修改/etc/hosts文件

      • 修改每台电脑的hosts文件。

        hosts文件和windows上的功能是一样的。存储主机名和ip地址的映射。

        在每台linux上，sudo vim /etc/hosts 编写hosts文件。将主机名和ip地址的映射填写进去。

      实现SSH免密钥登陆

      • 无密码登陆，效果也就是在naster上，通过 ssh node1 或 ssh node2 就可以登陆到对方计算机上。而且不用输入密码。
      • CentOS默认没有启动ssh无密登录，去掉/etc/ssh/sshd_config其中2行的注释，每台服务器都要设置，
      • RSAAuthentication yes

      • PubkeyAuthentication yes

      上传HADOOP并配置

      • 解压：

          HADDOP就在/opt/hadoop/hadoop-2.6.0里了

      • 配置：

        • 修改环境变量，将hadoop加进去（最后三个linux都操作一次）

          修改完后，用source /etc/profile配置文件生效。

      配置HADOOP文件

      • 配置之前，先在本地文件系统创建以下文件夹：~/hadoop/tmp、~/dfs/data、~/dfs/name。
      • 主要涉及的配置文件有7个：都在/hadoop/etc/hadoop文件夹下，可以用gedit命令对其进行编辑。

      ~/hadoop/etc/hadoop/hadoop-env.sh
      ~/hadoop/etc/hadoop/yarn-env.sh
      ~/hadoop/etc/hadoop/slaves
      ~/hadoop/etc/hadoop/core-site.xml
      ~/hadoop/etc/hadoop/hdfs-site.xml
      ~/hadoop/etc/hadoop/mapred-site.xml(需要COPY模板)
      ~/hadoop/etc/hadoop/yarn-site.xml

      • 进去hadoop配置文件目录

        cd etc/hadoop/

      • 配置 hadoop-env.sh文件–>修改JAVA_HOME

      • 配置 yarn-env.sh 文件–>>修改JAVA_HOME

      • 配置slaves文件–>>增加slave节点

        node1
        node2

      • 配置 core-site.xml文件–>>增加hadoop核心配置（hdfs文件端口是9000、file:/opt/hadoop/hadoop-2.6.0/tmp、）

      • 配置 hdfs-site.xml 文件–>>增加hdfs配置信息（namenode、datanode端口和目录位置）

      • 配置 mapred-site.xml 文件–>>增加mapreduce配置（使用yarn框架、jobhistory使用地址以及web地址）

      • 配置 yarn-site.xml 文件–>>增加yarn功能
